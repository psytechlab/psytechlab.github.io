# How we searched and how we chose
It all started when we needed to measure the prediction quality of different LLMs on one dataset. Besides the LLMs themselves, we also needed to compare basic prompt variants: zero-shot, few-shot, and chain-of-thoughts. When we started comparing, the question arose: how do we store and track all this? One of the task performers did everything in a spreadsheet, but it quickly became clear that this was a dead end: files get lost, there's no unified format, and filling them manually is tedious.

The idea came to look for some tool for tracking prompts and the LLMs they were run with. Surely we weren't the first to face this problem. Google confirmed this: such solutions exist and are called prompt management systems.

We defined the following list of criteria for selecting a platform:

* ability to deploy on your own server (self-host) — in a political storm situation, you need to carry everything with you, because there's a high risk that access to the service might be blocked from one side or another.
* collaboration — since we're a team, it's important that each team member can interact with the platform and be in the common context of what's happening.
* prompt versioning — if a prompt changes within one context, it should be versioned. Here it is — Git for prompts.
* adding metadata — ability to specify the metric that was obtained with this prompt, on which dataset it was applied, with which model, etc.
* integration with LLM frameworks — for greater convenience.

# A bit of terminology
The basic concept of Langfuse is a **trace**. Each trace is a representation of a request or operation that encompasses input and output data, as well as metadata. Each trace includes multiple **observations** — logs of individual execution steps. Traces can be grouped into **sessions**. A simple example of a session is a chat. Finally, there are **scores**. They are linked to traces and store the results of various measurements.

The important methods we will examine will be called exactly that, so it's useful to at least somewhat understand what they are and how they relate to each other.

# Working with Python
We will need this set of imports:

```python
import os
from openai import OpenAI
from langfuse.decorators import observe, langfuse_context
from langfuse import Langfuse
```

Before starting work, it's necessary to initialize Langfuse. To do this, you need to set the keys and initialize the client object:

```python
os.environ["LANGFUSE_PUBLIC_KEY"] = ""
os.environ["LANGFUSE_SECRET_KEY"] = ""
os.environ["LANGFUSE_HOST"] = "http://your.langfuse.ru:3000"

langfuse = Langfuse()

# Checking the correct setup of langfuse
langfuse.auth_check()
```

Now let's move directly to working with prompts. In Langfuse there are two types of prompts:

- **Text prompt** — this is an instruction in the form of a text string that is passed to the model to generate a single response.
- **Chat prompt** — this type is used for dialogue modeling. The main difference is that it includes conversation history and context.

Here's how prompt creation looks in code:

```python

# Creating a text prompt
langfuse.create_prompt(
    name="test_1",
    type="text",
    prompt='Write the capital of France',
    labels=["production"],
    tags = ['capitals'],
    config={
        "supported_languages": ["en", "ru"],
    },
)

# Creating chat prompt
langfuse.create_prompt(
    name="test_2",
    type="chat",
    prompt=[
      { "role": "system", "content": "You are a geographer" },
      { "role": "user", "content": "You need to respond only with the name of the capital of this country: {{country}}" },
    ],
    labels=["production"],
    config={
        "supported_languages": ["en", "ru"],
    },
)
```

As you can see, we pass the prompts in the "prompt" parameter. Let's remind ourselves about roles in prompts. There are three in total:

-	**User** – this is the user side interacting with the model. It contains requests, questions, instructions, or input data. For example, "Name the capital of Spain".
-	**System** – a kind of global model setting. Here you set the general context of the dialogue or task, specify how the model should behave, what style or format to use. For example, "You are a professional geographer giving a lecture at the world's best university."
-	**Assistant** – this is the role of the model itself. Here you specify what the response from the model should be like.

You may have also noticed that the chat prompt contains the syntax "{{country}}". This is a template variable. We can use it in both types. This variable will be replaced with a specific value that we pass when executing the request. This is very convenient as it allows us to form requests dynamically. We'll see exactly how later.

In addition to the "prompt" parameter and prompt type, we must give it a name. We'll retrieve the needed prompt from the server by name. We also passed some additional optional parameters:

-	**labels** - These are labels that allow you to classify or group prompts. If you create a new prompt by calling the `create_prompt` method with the same name and without a label, it will be assigned the `latest` label. By the way, globally the prompt version is handled by a special `version` field that contains an integer.
-	**tags** - These are additional keywords that help in organizing prompts and for searching in the web interface. Used to simplify search, analysis, and organization of requests.
-	**config** - This is a set of settings that specify request parameters or model behavior. Here are some parameters: supported_languages - supported languages; model — model name; temperature — roughly speaking, the degree of creativity; max_tokens — maximum number of tokens.

To get a request from the Langfuse server, you need to call the `get_prompt` method. There are two rules:
1. If you don't specify a `label`, you'll get the prompt version with the `production` label.
2. If you don't specify the chat `type`, the default type will be considered as `text`.

Next, we need to assemble the prompt using the `compile` method. This is where we can substitute values into the template variables that exist in our prompts. Keep in mind that without compilation, the output prompt will still contain the template. This can even be used intentionally, as long as it's conscious.

Here's how it will look for our prompts that we created above:

```python
prompt = langfuse.get_prompt("test_1", label='production')

compiled_prompt = prompt.compile()
print(f'Text prompt: {compiled_prompt}')

prompt = langfuse.get_prompt("test_2")

chat_prompt = langfuse.get_prompt("test_2", type="chat") 
compiled_chat_prompt_with_country = chat_prompt.compile(country='Spain')
print(f'Chat prompt (with substituted variable): {compiled_chat_prompt_with_country}')
compiled_chat_prompt = chat_prompt.compile()
print(f'Chat prompt (without substituted variable)- {compiled_chat_prompt}')

# >>> Text prompt: Write the capital of France
# >>> Chat prompt (with substituted variable): [{'content': 'You are a geographer', 'role': 'system'}, {'content': 'You need to answer only with the name of the capital of this country: Spain', 'role': 'user'}]
# >>> Chat prompt (without substituted variable) - [{'content': 'You are a geographer', 'role': 'system'}, {'content': 'You need to respond only with the name of the capital of this country: {{country}}', 'role': 'user'}]
```

# Request Logging
Now let's see how to make Langfuse monitor requests to the model. For this, you just need to define a function for the request and wrap it with the `observe()` decorator. It monitors the execution of the request and sends all information to Langfuse. Here's the simplest version of such a function for a text prompt:

```python
client = OpenAI(
  api_key='API_KEY',
  base_url='BASE_URL'
)

@observe()
def run_my_custom_llm_app(prompt_text):
    prompt = [{
        'role': 'user',
        'content': f'{prompt_text}'
    }]
    completion = client.chat.completions.create(
        model="gpt-3.5-turbo",
        messages=prompt,
        max_tokens=100,
    ).choices[0].message.content

    return completion

prompt = langfuse.get_prompt("test_1")
print(f'Query text - {prompt.compile()}')
output = run_my_custom_llm_app(prompt.compile())
print(f'AI response - {output}')

# >>> Query text - Write the capital of France
# >>> AI Response - Paris
```

For chat prompts, it differs very little. We only remove the creation of a list with objects, because the needed data format is already created by us from the chat prompt.
```python
@observe()
def run_my_custom_llm_app(prompt):
  messages = prompt
  completion = client.chat.completions.create(
      model="gpt-3.5-turbo",
      messages=messages
  ).choices[0].message.content

  return completion

prompt = langfuse.get_prompt("test_2")
compiled_prompt = prompt.compile(country='Spain') # We substitute the necessary value instead of the country variable
output = run_my_custom_llm_app(compiled_prompt)
print(f'Request - {compiled_prompt}')
print(f'AI Response - {output}')

# >>> Request - [{'content': 'You are a geographer', 'role': 'system'}, {'content': 'You need to answer only with the name of the capital of this country: Spain', 'role': 'user'}]
# >>> AI Response - Madrid
```

# Working with datasets
A very useful feature turned out to be the ability to create datasets within Langfuse. We were already prepared to suffer and manually handle tracking which dataset experiments are run on, but it wasn't necessary. Of course, there's some redundancy - on one hand, ClearML is our main storage, on the other hand, we'll be duplicating some datasets in Langfuse. We decided we'll live with this for now, defining two rules. First, all datasets in Langfuse will be secondary to datasets from ClearML. Second - always specify the dataset ID from ClearML in the Langfuse dataset metadata, so we can always trace where the latter's megabytes are coming from. 

To create a dataset, it's enough to execute the following command:

```python

langfuse.create_dataset(name='Capitals', description="Optional description", metadata={"clearml_id": "CLEARML_ID"})
```

# Experiments
Let's now imagine that we're interested in how well the LLM knows the capitals of different countries. We've prepared the test dataset, we just need to decide on a metric, run the dataset through the LLM and measure the results. As a starting point, we'll use `test_2`, which is chat-based. For requests, we'll use the latest version of the `run_my_custom_llm_app` function.

As a metric, we'll take the simplest option — we'll just compare what the LLM returns with what we expect from it:

```python
def simple_evaluation(output, expected_output):
  return output == expected_output
```

We need this for demonstration purposes. Actually, it would be better to use Levenshtein distance. You'll understand why later. Now we need to prepare a function to run the experiment. Here's what it looks like:
```python
def run_experiment(experiment_name, system_prompt):
  dataset = langfuse.get_dataset("Capitals")

  for item in dataset.items:
    # item.observe() returns trace_id, which can be used to add custom evaluations later
    # automatic linking of tracing with experiment run
    with item.observe(run_name=experiment_name) as trace_id:
      output = run_my_custom_llm_app(system_prompt.compile(country=item.input['country']))
      print(output)
      # adding evaluation results to experiment trace (simple_evaluation)
      langfuse.score(
        trace_id=trace_id,
        name="exact_match",
        value=simple_evaluation(output, item.expected_output)
      )
```

The `item.observe(run_name=experiment_name)` method returns trace_id, where all observations of the `run_my_custom_llm_app` function and measurements from the `langfuse.score()` call are stored. You can define as many measurements as you need. There's an option to connect LLM-as-a-judge, more details about this are available [here](https://langfuse.com/docs/scores/model-based-evals)

The annotation schema needs to be registered in the system. To do this, go to the "Settings -> Scores/Evaluation" tab. Annotation can have three types:

*   Numeric: Used for scale-based evaluation (e.g., from 1 to 10). Applied for measuring degree of correspondence, relevance, or other quantitative parameters.
*   Categorical: Represents selection from predefined categories (e.g., "Positive", "Negative", "Neutral"). Convenient for classifying responses.
*   Boolean: Designed for simple "yes/no" evaluations. Useful for checking binary conditions (e.g., "correct/incorrect answer").

# Managing and preserving context in chats
Finally, we'll show how to create a dialog with saving the model's response for context. This example demonstrates a basic implementation of a chat-bot with context preservation and dynamic interaction. We preserve context by adding each new message to the chat history. Thus, with each request, the model receives the full dialog, including previous messages, which allows it to form a response taking into account the entire conversation context. It should be understood that token consumption will be higher in this case, as all past messages will be taken into account.

```python

# Client Initialization
client = OpenAI(
    api_key='',
    base_url=''
)

# Message History
chat_history = [
    {"role": "system", "content": "You are a bot that answers questions about geography. Answer briefly and clearly."}
]

@observe()
def run_my_custom_llm_app(user_input):
    global chat_history

    # Add user message to history
    chat_history.append({"role": "user", "content": user_input})

    # Make request to model
    completion = client.chat.completions.create(
        model="gpt-3.5-turbo",
        messages=chat_history
    ).choices[0].message.content

    # Add model response to history
    chat_history.append({"role": "assistant", "content": completion})

    return completion

prompts = ['Name the capital of Russia', 'Tell me briefly about this city'] # prompts to continue the session
for i in prompts:
  response = run_my_custom_llm_app(i)
  print(f'You: {i}')
  print(f'AI: {response}')

# >>> You: Name the capital of Russia
# >>> AI: Moscow.
# >>> You: Tell me briefly about this city
# >>> AI: Moscow is the capital of Russia, the largest city in the country and one of the largest cities in the world by population. Founded in the 12th century. Moscow is the political, economic and cultural center of Russia. The city contains numerous historical monuments, museums, theaters and other attractions.
```
